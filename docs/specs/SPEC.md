# Spec: ChadGPT — Chadchart Policy Chat Agent

## Objective

Build a Thai-language chat agent that lets Bangkok residents ask natural questions about Chadchart Sittipunt's policies and progress. The agent retrieves relevant content from pre-embedded policy/progress data and generates a grounded Thai answer via an LLM.

**Primary users:** Bangkok residents — both undecided voters and existing supporters seeking shareable, specific answers.

**Success criteria (30 days):**
- A user receives a specific, accurate Thai answer in under 5 seconds
- At least one answer is shared on social media as a screenshot
- Monthly infrastructure cost stays under $20 (target: $0 on free tiers)
- The team can demo it internally on Vercel within one week

---

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Web UI + Vercel API | Next.js 14 (App Router) | Free Vercel hosting, API routes for demo |
| Language | TypeScript | Type safety across engine + adapters |
| Embeddings (offline) | Gemini `text-embedding-004` | Free tier, 768-dim unit vectors |
| LLM (runtime) | Gemini 2.0 Flash (default) | Free tier (1,500 req/day) |
| LLM alternatives | Claude Haiku, OpenAI GPT-4o-mini | Switchable via env var |
| Search | In-memory cosine similarity | No database, no cost |
| Cloudflare adapter | Cloudflare Workers (future) | Edge deployment after team approval |

## Core Interfaces (`src/engine/types.ts`)

```typescript
// A single item from the policy or progress JSON files, with its embedding baked in
interface PolicyItem {
  text: string;          // The searchable text (policy_description or progress_text)
  category: string;      // Thai category name
  subcategory?: string;  // Only present in policy files
  source_url: string;
  source_file: string;   // e.g. "progress_people", "policy_city"
  embedding: number[];   // 768-dim unit vector from text-embedding-004
}

// Result of semantic search
interface SearchResult {
  item: PolicyItem;
  score: number;         // Cosine similarity 0–1
}

// LLM provider abstraction
interface LLMProvider {
  generate(systemPrompt: string, userMessage: string): Promise<string>;
}

// API request/response
interface ChatRequest {
  message: string;
}

interface ChatResponse {
  answer: string;
  sources: { category: string; source_url: string }[];
}
```

## LLM Provider Design (`src/engine/providers/`)

Selected at startup via `LLM_PROVIDER` env var. Default: `gemini`.

```typescript
// src/engine/providers/index.ts
function createProvider(env: ProviderConfig): LLMProvider {
  switch (env.LLM_PROVIDER) {
    case 'claude':  return new ClaudeProvider(env.CLAUDE_API_KEY);
    case 'openai':  return new OpenAIProvider(env.OPENAI_API_KEY);
    default:        return new GeminiProvider(env.GEMINI_API_KEY);
  }
}
```

Each provider implements the same interface. Swapping LLMs requires changing one env var only.

## Embedding Generation Script (`scripts/embed.ts`)

**Runs once, offline. Not part of the runtime server.**

Behaviour:
1. Read all 8 source JSON files from `docs/Chadchart Policy/`
2. For each item, build a combined text: `หมวดหมู่: {category}\n{description or progress_text}`
3. Call `text-embedding-004` via Gemini API (200ms delay between calls to respect rate limits)
4. Write output to `data/embedded/{filename}.json` with `embedding` field added
5. Log progress: `✓ [source_file] item N/total`

Output schema per item — same as source JSON plus:
```json
{
  "embedding": [0.023, -0.147, ...],
  "source_file": "progress_people"
}
```

The `data/embedded/` files are **committed to the repo**. The script only needs to re-run when source data changes.

## Semantic Search Logic (`src/engine/search.ts`)

```typescript
const TOP_K = 5;
const MIN_SCORE = 0.5;

async function semanticSearch(
  question: string,
  db: PolicyItem[],
  embedFn: (text: string) => Promise<number[]>
): Promise<SearchResult[]> {
  const queryVec = await embedFn(question);

  return db
    .map(item => ({ item, score: dotProduct(queryVec, item.embedding) }))
    .filter(r => r.score >= MIN_SCORE)
    .sort((a, b) => b.score - a.score)
    .slice(0, TOP_K);
}

function dotProduct(a: number[], b: number[]): number {
  let sum = 0;
  for (let i = 0; i < a.length; i++) sum += a[i] * b[i];
  return sum;
}
```

If no results pass `MIN_SCORE`, return a Thai "I don't know" response without calling the generation LLM — saves API quota.

## Chat Orchestration (`src/engine/chat.ts`)

```
User question
  → Embed question (Gemini text-embedding-004)
  → Semantic search (top 5, score ≥ 0.5)
  → If no results: return polite "ไม่พบข้อมูลที่เกี่ยวข้อง" message
  → Build context string from top results
  → Call LLM provider with system prompt + context + question
  → Return { answer, sources }
```

System prompt (Thai):
```
คุณคือผู้ช่วยตอบคำถามเกี่ยวกับนโยบายและผลงานของชัชชาติ สิทธิพันธุ์ ผู้สมัครผู้ว่าราชการกรุงเทพมหานคร
ตอบเป็นภาษาไทยเท่านั้น กระชับ ชัดเจน และใช้เฉพาะข้อมูลที่ได้รับมา
ถ้าไม่มีข้อมูลที่เกี่ยวข้อง ให้บอกว่าไม่ทราบ อย่าเดา
```

## Web UI

Single-page chat interface. Minimal, functional. Thai-first.

**Features:**
- Chat window with message history (user + assistant bubbles)
- Input field + send button (Enter to send)
- Loading state while waiting for response
- Sources listed below each assistant answer (category + link)
- Suggested starter questions in Thai (shown before first message):
  - "ชัชชาติทำอะไรด้านสุขภาพไปแล้วบ้าง?"
  - "นโยบายด้านการศึกษามีอะไรบ้าง?"
  - "แก้ปัญหารถติดยังไง?"

**Not doing (MVP):**
- Multi-turn memory (each question is independent)
- User accounts or history persistence
- Thai keyboard optimization
- Dark mode

## Data Flow Summary

```
[Offline — run once]
scripts/embed.ts
  → reads docs/Chadchart Policy/**/*.json (8 files)
  → calls Gemini text-embedding-004
  → writes data/embedded/**/*.json (committed)

POST /api/chat { message }
  → src/adapters/vercel.ts
  → src/engine/chat.ts
      → embed question (text-embedding-004)
      → cosine search against in-memory db
      → if no match: return Thai fallback
      → build context from top 5
      → call LLM provider
  → return { answer, sources }
```

## Boundaries

**Always:**
- Keep `src/engine/` free of Node.js-only APIs (`fs`, `path`, `process` are allowed only in `scripts/`)
- Validate `message` field in API route before calling engine (non-empty, max 500 chars)
- Never send raw embeddings to the client
- Commit `data/embedded/` JSON files — they are build artifacts, not secrets

**Ask first:**
- Changing `MIN_SCORE` or `TOP_K` (affects answer quality)
- Adding a new LLM provider
- Changing the system prompt Thai text
- Adding Line OA or web chat channel adapters

**Never:**
- Commit `.env.local` or any file containing API keys
- Call the LLM if semantic search returns zero results (wastes quota, gives hallucinated answers)
- Load source data at request time (always use pre-embedded files)
